组件层 · 用法

每个组件只定义一次：样式在 <style> 的对应 CSS 块、结构在底部 COMPONENT LAYER 脚本。页面任何位置（含组件互相嵌套）一律写 <g3d-*> 占位标签，不再手抄内部 DOM——避免同一组件在多处被各自"自定义"而走样，也让"用到哪些组件、组件如何区分"在标记里一目了然。 占位标签在加载时就地展开为规范 class 标记，故查看源码=看组件用法，审查 DOM=看规范结构，且 1:1 对应 C++ 侧 G3DWidgets。

字体层级

常规（基准）字号 = 14px，正文与主要控件（按钮 / 输入 / 菜单 / 标签页 / 树行）统一用它。 这是业内成熟规范对专业工具的"常规"取值：Ant Design 14、Microsoft Fluent 14、Material 14； macOS HIG 13pt、VS Code ~13 偏 Latin IDE，Figma/Blender ~11 偏超密集创作工具——CJK 界面取 14 比 13 更耐读。桌面端（ImGui）同步为 14px。

各场景统一吗？不完全统一——常规是 14px，其余按"为什么"刻意分级：标题放大利于扫读，次要 / 分组信息缩小提升密度，超密集数据行降一级以容纳更多内容。

Title / 20操作面板 Inspector页面 / 分区标题
Subtitle / 16材质 Material面板 / 对话框标题
Base / 14 ★常规：正文 + 按钮 / 输入 / 菜单 / 标签页 / 树行统一基准
Caption / 12次要：字段标签 / 提示 / 数值 / 计数 · 紧凑控件（分段）次要信息 / 密集控件降一级
Overline / 11SECTION TITLE 分组标题大写 + 字距，弱化为结构标识
Dense / 13超密集数据行（树 / 表 dense 模式）为容纳更多行降一级
令牌见 :root 的 --fs-title / --fs-subtitle / --fs-base / --fs-caption / --fs-overline / --fs-dense； 组件只引用令牌，改基准改这一处。注：本规范主字体为单字重 MonaspaceNeon，层级靠字号 + 颜色 + 间距区分，不依赖粗体。

组合图标按钮 g3d-iconbtn = 仅含一枚 图标 g3d-icon 的按钮容器。

输入框 / 选择

聚焦 = 主色描边 + 焦点环 + 底色微亮；支持前置图标、占位符、错误态、下拉。

开关 / 复选 / 单选 / 分段

开关/复选/单选可单独用，也常与文字标签成行（switchrow）。分段是独立的单选条。

组合带标签整行 g3d-switchrow = g3d-toggle / g3d-check / g3d-radio + 文本；其中 g3d-check 内含 图标 g3d-icon（对勾）。g3d-segmented 不依赖其它组件。

树 / 大纲 ≈ImGui Clipper

面向专业编辑器的场景大纲（Outliner）：默认紧凑高密度、整行选中、缩进引导线、悬停行内操作、可见性/锁定开关、拖拽排序。 借鉴 VS Code Outline、Blender Outliner、Figma 图层。大批量节点靠固定行高 + 虚拟化（仅绘制可视行）支撑。

组合每行 g3d-trow 复用 图标 g3d-icon（类型 / 折叠箭头 / 行内操作）与 复选 g3d-check（三态批量选）；工具栏复用 输入框 g3d-input + 图标按钮 g3d-iconbtn，密度切换复用 分段 g3d-segmented。
